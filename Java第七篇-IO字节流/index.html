2.IO概述分类

1.概念

  • IO流用来处理设备间的数据传输（相当于管道）
  • Java对数据的操作室通过流的方式
  • 操作流的类都在IO包中
  • 流操作分为两种：
    字节流：可以操作任何数据
    字符流：只能操作纯字符数据 ，操作字符数据方便（例如各种文字）

  • 字符
    InputStream抽象类：字节输入流所有类的超类
    OutputStream抽象类
  • 字符流
    Reader
    Writer

    IO程序书写

    使用前，导IO包
    使用时，进行异常处理
    使用后，释放资源

读取示例

int read() 从输入流中读取一个数据字节

当有效字符读完后，再读就会读到-1（每种文件结尾标记不一样）

为什么不用byte用int接受

  1. 首先，计算机存储都是用补码存储的（正数补码=原码，负数除了符号位取反+1）：
    因为cpu只有加法器，用补码存储可以把减法统一到加法中，而且0的补码唯一（0和-0的补码一样）
  2. 所以如果用byte接收，中间可能有个字节是11111111（byte型-1的补码），没读完程序就会停止
  3. 如果用int，读每个字节都会在前面补24个0，凑够4个字节，所以如果读到11111111，type的-1就变成了int的255，这样可以保证读完，结束标记的-1是int型的

写入示例

  • 构造函数 BufferedInputStream(InputStream in)

    创建缓冲区对象，对输入流包装，让其更强大

  • 缓冲区默认长度是8192，底层使用数组完成的。
    缓冲区一次从原文件读8192个字节，然后java程序read()一个个字节的从缓冲区读，不用经过硬盘
    程序再一个个的写到输出缓冲区，写到8192个字节后，一次写到目标文件z

flush()和close()

  • 刷新：
    在缓冲区最后一次可能没存满8192个字节，就不写了，而刷新，就会把剩下的不满8192个字节的数据，也写进去

7.IO流标准处理异常代码

  • 首先，不能直接抛出异常就不管了，因为六句代码都有可能有异常，如果第一句就有异常，后面代码不执行，流就不会关闭，所以要用try..finally
  • 还有，要在try的前面定义流，并且赋值null,因为如果没有初始化，在给流赋值的时候如果失败的话（比如文件丢失），流的引用就是空的，就不能调用close()
  • 给fis,fos赋值为null后，如果读写失败的话，还是=null,调用close()会空指针异常，所以调用close()要判断fis,fos是否为null
  • 最后，有可能关输入流的时候异常，就不能关输出流了 ，所以要有finally嵌套
